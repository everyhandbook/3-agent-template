# lessons

이 문서는 프로젝트 운영 중 반복되는 교훈, 실수 방지 포인트, 다음 세션에 도움이 되는 체크리스트를 누적하는 기본 문서입니다.

## 왜 남기는가

- 같은 실수를 반복하지 않기 위해
- 다음 세션이나 다른 agent가 빠르게 맥락을 이어받기 위해
- 운영 규칙이 실제 작업에서 어떻게 쓰였는지 축적하기 위해

## 언제 업데이트하는가

- 세션 종료 시 `session-wrap-up`을 수행할 때
- checkpoint 없이도 중요한 운영 교훈이 생겼을 때
- sync, handoff, 배포, 문서화 과정에서 반복 패턴이 드러났을 때

## 어떤 내용을 남기는가

- 반복되는 실수와 방지 방법
- 다음 세션에서 먼저 확인해야 할 체크포인트
- 특정 도구나 workflow를 쓸 때 유용했던 패턴
- 문서 구조나 운영 방식에서 개선된 점

## 작성 원칙

- 짧고 명확하게 씁니다.
- 확정된 교훈만 남기고, 추측은 따로 표시합니다.
- 특정 세션 로그와 연결되면 관련 파일 경로를 함께 남깁니다.

## 초기 메모

- 새 skill을 만들거나 크게 수정할 때는 먼저 `skill-creator` 시스템 skill을 확인합니다.
- 항상 켜져야 하는 운영 정책은 skill보다 rule이 더 맞는지 먼저 판단합니다.
