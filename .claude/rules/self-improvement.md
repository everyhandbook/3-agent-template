# 자기 개선 규칙

이 템플릿은 반복 가능한 실수나 운영 교훈을 `docs/lessons.md`에 누적해 다음 세션과 다른 agent가 재사용하도록 설계합니다.

## 언제 기록하는가

- 사용자 피드백으로 작업 방식이나 응답 톤을 수정했을 때
- 같은 실수나 혼동이 다시 나올 가능성이 높을 때
- 도구 설정, 동기화, handoff 같은 운영 함정을 발견했을 때
- 규칙이나 workflow를 바꾼 이유를 다음 세션에도 알아야 할 때

## 어디에 남기는가

- 반복 가능한 운영 교훈: `docs/lessons.md`
- 특정 세션에만 유효한 메모: `docs/session-logs/`

## 기록 원칙

- 교훈은 짧고 실행 가능하게 적습니다.
- 단순 결과만 쓰지 말고, 왜 문제가 생겼는지와 다음부터 어떻게 막을지를 함께 남깁니다.
- 아직 확정되지 않은 추측은 lesson이 아니라 세션 로그에 남깁니다.

## 세션과의 연결

- `session-start`는 최근 `docs/lessons.md` 항목을 함께 확인합니다.
- `checkpoint`와 `session-wrap-up`은 새로운 교훈을 남겨야 하는지 함께 판단합니다.

## 권장 형식

```md
## YYYY-MM-DD

- 상황:
- 문제:
- 해결:
- 다음부터는:
```
