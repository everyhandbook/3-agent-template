# agent-handoff

다른 agent에게 작업을 넘기거나, 검토 결과를 전달하거나, 현재 상태를 빠르게 공유해야 할 때 사용하는 범용 skill입니다.

가능한 한 채팅 장문 대신 레포 안의 `.md` 문서로 먼저 정리합니다.

## 문서 유형

상황에 따라 아래 유형 중 하나를 선택합니다.

- `brief`
  - 다른 agent에게 새 작업을 요청할 때
- `review`
  - plan이나 구현 결과를 검토한 결과를 전달할 때
- `handoff`
  - 현재 상태와 남은 일을 다른 agent에게 넘길 때
- `status`
  - 짧은 진행 상황만 공유할 때

의도가 모호하면 가장 최근 대화 맥락에서 가장 적합한 유형을 선택합니다.

## 저장 위치 규칙

기본 저장 위치는 아래를 사용합니다.

- `docs/session-logs/YYYY-MM-DD_[topic]-[type].md`

예시:

- `docs/session-logs/2026-04-15_template-brief.md`
- `docs/session-logs/2026-04-15_template-review.md`

사용자님이 특정 경로를 지정하면 그 경로를 우선합니다.

## 공통 작성 원칙

1. 상대 agent가 바로 이어서 판단하거나 실행할 수 있어야 합니다.
2. "무엇을 했는가"보다 "지금 무엇이 live 상태인지 / 무엇이 남았는지"를 더 분명히 씁니다.
3. 미확정 내용은 결정처럼 쓰지 않고 `가정`, `확인 필요`, `미확정`으로 표시합니다.
4. 다음 액션 또는 검토 포인트를 반드시 남깁니다.
5. 필요하면 관련 파일 경로를 함께 적되, 문서 본문은 과하게 길게 늘리지 않습니다.

## 작성 절차

1. 관련 문서를 먼저 확인합니다.
   - `task.md`
   - 관련 `README.md`
   - 이번 세션에서 수정한 파일
   - `docs/session-logs/`의 관련 로그
   - 필요하면 `docs/lessons.md`
2. 문서 유형을 정합니다.
3. 아래 템플릿에 맞춰 문서를 저장합니다.
4. 채팅에는 저장 경로와 핵심 요약만 짧게 공유합니다.

## 템플릿

### brief

```md
# [주제] 브리프

## 작업 내용

## 참고해야 할 파일

## 기대 산출물

## 주의사항

## 다음 액션
```

### review

```md
# [주제] 검토 결과

## 검토 대상

## 판정

## 적절한 점

## 보완 필요 사항

## 다음 액션
```

### handoff

```md
# [주제] 핸드오프

## 목적

## 현재 결론

## 현재 live 상태

## 남은 일

## 검토 포인트

## 관련 파일
```

### status

```md
# [주제] 상태 업데이트

## 현재 상태

## 완료된 것

## 진행 중인 것

## 다음 액션
```

## 주의사항

- 기존 세션 로그와 겹치면 새 파일 생성보다 보완 업데이트가 더 자연스러운지 먼저 확인합니다.
- 단순 changelog가 아니라, 다른 agent가 바로 이어서 이해할 수 있는 문서가 되도록 씁니다.
- 구현이 끝난 뒤에도 남은 리스크가 있으면 숨기지 않습니다.
